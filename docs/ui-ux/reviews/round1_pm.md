## HIGH 指摘 (5件) — リリース前に必ず解消

### H1. North Star Metric / Activation Metric が未定義
- **該当箇所**: §1, §16
- **問題**: プロダクト原則は4つあるが「**この数値が伸びれば事業が伸びる**」という North Star が無い。F-01 の成功条件は「完了率70%」「中央値9分」「LinkedIn接続≥1」とプロセスKPIのみで、 **アクティベーション (= ユーザーが価値を実感した瞬間)** が定義されていない。HeyReach / Expandi では一般に「初回返信を受け取った日」が Aha Moment として知られる。
- **推奨**:
  - North Star: `週次の Reply received per active LinkedIn account` を採用 (会社の売上に最も連動)
  - Activation Metric: `Sign-up → 7日以内に最初の REPLIED 状態が発生`
  - §16.1 にイベント `activation.aha_moment_reached` を追加し、§5.1 オンボ成功条件にも反映
  - ダッシュボード S03 に「Aha まで残り◯日 / 残り◯リード」のオンボ進捗バナーを Day 0–14 で表示
- **期待効果**: D7 リテンション +15〜25pt、トライアル → 有料転換率 +5〜10pt (業界ベンチマーク: HubSpot, Outreach の Activation 設計に準拠)

### H2. 差別化が「日本語が綺麗」+「安全」止まりで Defensible Moat になっていない
- **該当箇所**: §1.1, §21.4 競合表
- **問題**: §21.4 の差別化表は内向き (社内納得用) であり、 **顧客が他社から乗り換える理由 (PoD = Point of Difference) が弱い**。HeyReach は$79/月から、Expandi は$99/月、La Growth Machine は €120/月で機能成熟。日本語が綺麗だけでは ¥48,000/月 (Growth) は割高に見える。「会社特化 RAG」も実体が§13に2画面しかなく、価値が伝わらない。
- **推奨**:
  - PoP (Point of Parity): HeyReach と同等のコネクト/DM/受信箱/CRM
  - PoD (Point of Difference) を3つに絞り込む
    1. **日本語ICP × Sales Navigator 検索式の自動生成** (英語SaaSがどうしても弱い領域)
    2. **会社特化 RAG ("根拠を見る") の出典ハイライト** — 単なるテンプレ管理ではなく、LP/事例PDF/IR資料が文章にどう影響したかを可視化
    3. **HITL レビューモードの段階的緩和** — 自動送信に至るまでの "信頼ビルダー" UX (3通連続採用ログ等)
  - LP / マーケコピー / セールスデッキ / プロダクト内 onboarding の3カ所で同じPoDが反復されるよう、§5.1 オンボフロー第1画面に PoD ステートメントを配置
  - §21.4 の比較表に **価格** と **日本語サポート品質** の2列を追加
- **期待効果**: 受注率 +20% (HeyReach比較で「なぜ我々を選ぶか」が言語化される)、CAC低下

### H3. プライシング設計が "Growth ¥48,000/月" 1プラン提示のみで商業的に致命
- **該当箇所**: §6.8 S14 プラン/請求
- **問題**:
  - **Good/Better/Best (3プラン構造)** が無く、Anchor Pricing が機能しない → ユーザーは比較対象が無いと「高い」と感じる
  - **年契約割引** (通常2ヶ月分OFF) の UX が無い → ARR 化と churn 抑制の最大武器を放棄
  - **無料トライアル / Freemium / Self-serve PoC** の入口が不明 → SLG (Sales-Led) しか取れず、CAC が膨らむ
  - **ダウングレード / 一時休止 (Pause Plan)** UX が無い → SaaS チャーンの30%以上を占める "今月だけ忙しい" 顧客を逃す
  - **チームシート vs LinkedIn接続数** の課金軸が不明 (5接続=¥48,000?)
- **推奨**:
  - プラン構造: Solo (¥9,800/月、1接続) / Team (¥48,000/月、5接続) / Scale (¥120,000/月、20接続+SSO+監査) の3階層
  - 14日無料トライアル (クレカ不要) → 期限切れ時は「Solo へ自動ダウングレード」(完全停止しない)
  - 年契約 -17% (= 2ヶ月分OFF) を S14 にトグルで提示、契約日と更新日を可視化
  - **休止プラン** (¥0/月、データ保持90日、送信停止) を S14 と退会フローに追加 → 退会の代わりに休止を提案
  - 上限到達時の「従量課金で続行」は **既定OFF** (現行設計) は良いが、 **超過予約** (今月だけ +500リードを ¥3,000 で買う) の UX を追加
- **期待効果**: ARR +25% (年契約化)、Net Churn -8pt (休止プラン化)、トライアル流入 +3倍 (無料トライアル化)

### H4. CRM 連携 (S12) が Phase 2 = 商業的に矛盾
- **該当箇所**: §4.1, §19, §6.6 S12
- **問題**: ペルソナP1 (営業マネージャ) のJTBD J4「商談化を逃さない → 即CRM連携」が **MVPで成立しない**。Sales Tech SaaS の購買決定者 (P1) は **HubSpot/Salesforce連携が無いツールは検討対象外**。Phase 2 (4ヶ月後) では商談化機会の半年〜1年を逃す。
- **推奨**:
  - **S12 を Phase 1 に昇格**、ただしスコープを「HubSpot 一方向 (Lead → Contact 作成のみ)」に絞る (Salesforce / Pipedrive は Phase 2)
  - HubSpot は API simple、 OAuth 1日で組める。MVP に必須
  - CRM 未接続ユーザーには「CSV エクスポート + Zapier 経由」のフォールバック導線を S12 に併記
  - もしくは Phase 1 のターゲットを **P3 (1人代理店)** に絞り、 P1 は Phase 2 リリースと同時に獲得開始するロードマップに変更 (Go-to-Market 整合性を取る)
- **期待効果**: P1 ペルソナの早期受注、ACV +¥120,000/年、商談獲得率 +30%

### H5. オンボ (F-01) のドロップオフ対策が "完了率 70%" 目標のみ、復帰導線が無い
- **該当箇所**: §5.1 F-01, §11
- **問題**:
  - 9分・5ステップは野心的 (一般的なB2B SaaSは15〜20分)。30%が脱落する前提なのに **未完了ユーザーへの復帰 UX が無い**
  - LinkedIn接続失敗時のリカバリ (Unipile OAuth エラー、CAPTCHA、ログイン拒否) が §5.1 に書かれていない
  - **Onboarding チェックリスト** (Asana / Linear 風) が S03 ダッシュに無い → 中断したユーザーが「次に何をすべきか」を見失う
- **推奨**:
  - S03 ダッシュ上部に Sticky な Onboarding Checklist (5タスク、進捗バー、24h以内に完了で特典) を Day 0–14 表示
  - 未完了ユーザーへ Day 1 / Day 3 / Day 7 のメール (Lifecycle Marketing)、 Day 7 で Owner 担当者からの 1:1 オンボ DM を提案
  - LinkedIn 接続失敗時の専用画面: Unipileエラーコード別に「具体的な対処手順 + サポートチャットボタン」
  - §5.1 に「離脱時挙動」を追記: ステップ離脱時は Draft 自動保存し、再ログイン時は離脱位置に復帰
- **期待効果**: オンボ完了率 70% → 85%、トライアル → 有料転換率 +10pt

## MEDIUM 指摘 (6件) — Phase 1 中に解消

### M1. ICP × ペルソナの ARPA / CAC / LTV が未定義
- **該当箇所**: §2 ペルソナ
- **問題**: P1〜P4 の定性記述はあるが、 **誰がいくらでどの位の頻度で買うか** の単価モデルが無い。プロダクトの最重要ペルソナ (= ICPの中のICP) が判別できない。
- **推奨**: 各ペルソナに `想定ARPA / 想定LTV / 想定CAC / 想定LTV:CAC比` を追記。例えば P3 (1人代理店) は LTV低・CAC安、P1 (VPoS) は LTV高・CAC高。プロダクトとマーケが同じ ICP を見ているか整合確認
- **期待効果**: GTM の優先順位が明確化、機能優先度の意思決定基準化

### M2. リテンション施策 (Habit Loop / Triggers) の設計が薄い
- **該当箇所**: §11 通知, §16 計測
- **問題**: 通知レベル (§11.1) は良いが、 **Hook Model (Trigger → Action → Variable Reward → Investment)** に基づく日次/週次の習慣化フックが弱い。ユーザーが毎朝アプリを開く理由が「返信があるから」だけでは脆い (返信が無い週=離脱)
- **推奨**:
  - Daily Digest メール: 毎朝 8:30 「昨日の活動 + 今日のおすすめアクション3つ」
  - Weekly Review: 月曜朝 9時に「先週のベストキャンペーン / 改善提案 / 業界平均比較」
  - Social Proof: 「同業界の平均 返信率 12% / あなた 18% (Top 20%)」
  - Streak: 連続稼働日数 (7日連続でバッジ)
- **期待効果**: WAU/MAU比 +10pt、 D30リテンション +12pt

### M3. プラン上限到達時の "Upgrade UX" が "停止" 一択で売上機会損失
- **該当箇所**: §6.8 S14
- **問題**: 現行設計「上限到達 = 既定で停止」は安全側で正しいが、 **アップグレード CTA が画面のどこにあるか不明**。送信停止後のユーザーは「上位プラン提案」を見ずに離脱する可能性
- **推奨**:
  - 上限到達 7日前 / 3日前 / 1日前 / 当日 のアプリ内通知 + メール
  - 上限到達時の停止画面に「今すぐ Scale プランで再開 (差額 ¥XX のみ)」CTA を最大化、二次CTA に「来月まで待つ」
  - **Pro-rated upgrade** (月途中アップグレード時の日割り計算) を可視化
- **期待効果**: プラン昇格率 +5〜8pt、月中拡張売上 +¥XXX,XXX

### M4. "返信率"・"承認率"・"商談化率" の業界ベンチマーク表示が無い
- **該当箇所**: §6.1 S03, §16.2 KPI数式
- **問題**: KPI 数式 (§16.2) と前期比 (§6.1) は良いが、 **「自分の数値が業界平均と比べて高いか低いか」** が可視化されない。これは Sales Tech SaaS の最重要価値の一つ (Outreach, Salesloft の中核機能)
- **推奨**:
  - 各KPIカードに業界中央値 / Top 10% を併記 (例: 返信率 18% / 業界平均 12% / Top 10% 22%)
  - 業界 × 役職 × 規模での セグメント別ベンチマーク (§13 ナレッジに入る匿名統計データから集計)
  - 「あなたは Top 20%」のバッジを §11.1 通知 Info レベルで配信
- **期待効果**: NPS +15、口コミ獲得増、Daily 開封率 +20pt

### M5. "ナレッジ" (§13) の価値仮説が抽象的、PoC で証明が必要
- **該当箇所**: §6.7 S13, §1.1 PoD
- **問題**: 「会社の声を AI に渡すための辞書」「品質バッジ (引用数 / 重複 / 古さ)」は良いが、 **これがどう商談化率を上げるか** の数値仮説が無い。RAG搭載は実装コストが高く、価値証明が無いと Phase 2 に倒れたとき Phase 3 にずれる
- **推奨**:
  - Phase 1 で 5社 のクローズドベータを実施し、「ナレッジ あり/なし で返信率 +X%」を検証
  - §13 に「期待される効果: ナレッジ導入で返信率 +5〜8pt (仮説)」を明記
  - ナレッジが空のユーザーには「サンプルナレッジ (業界別)」を提供し、ゼロから書かせない
- **期待効果**: ナレッジ機能の活用率 80%超、PoD の実証

### M6. 退職者ハンドオフ (F-05) の「退職者本人通知」と「移管期限」が未定義
- **該当箇所**: §5.5 F-05
- **問題**: ハンドオフフローは良いが、 **退職者本人にダッシュボード通知が飛ぶか** 、 **未移管リードの放置上限 (例: 7日後に自動アーカイブ)** が無い。実運用で「退職した田中さん名義のリードが3ヶ月放置」が発生する
- **推奨**:
  - 退職者本人にメール「あなたのアカウントは無効化されました。所属企業のデータは XX に引き継がれます」
  - 未移管リードの30日経過で自動エスカレーション (Owner にアラート)
  - F-05 に「P4 (情シス) 視点の チェックリスト UI」を追加
- **期待効果**: 大企業導入時の情シス審査通過率 +30%、データ滞留事故 ゼロ

## LOW 指摘 (4件) — 改善余地

### L1. 名称 "LinkdInside" のブランド検証が無い
- **該当**: §1 冒頭
- **問題**: LinkedIn 商標との衝突可能性 / SEO上の不利 / 発音しづらさ。「Linkd」は Sunset した類似サービス名と被る
- **推奨**: 法務調査 (商標) + 主要言語での発音テスト + ドメイン取得可能性確認 を実施。並行して 3〜5案のネーミング候補をユーザーテスト
- **期待効果**: 法的リスク低減、ブランド想起率 +20%

### L2. 顧客成功 (CS) チャネルの設計が無い
- **該当**: §11 通知
- **問題**: アプリ内チャット / Help Center / Slack Community / オンボ動画 の戦略が不明
- **推奨**: Phase 1 で Intercom or Crisp を組込み、 §6.x ヘルプ画面に Onboarding Video (90秒×5本) を配置。 Phase 2 で Slack Community
- **期待効果**: CS 工数 -40%、自己解決率 +25pt

### L3. プロダクト内 NPS / CSAT 計測の組込みが無い
- **該当**: §16
- **問題**: PostHog でイベントは取るが、 **満足度の定性指標** が無い。プロダクト改善の優先順位が事実ベースで決まらない
- **推奨**: 90日に1回の NPS アンケート (アプリ内モーダル)、商談化直後の CSAT (5段階)、退会フローでの離脱理由アンケート (必須選択式)
- **期待効果**: ロードマップ意思決定の精度向上、Detractor の早期発見

### L4. "招待によるバイラル成長" の機能が無い
- **該当**: §6.9 S15
- **問題**: メンバー招待は基本機能のみ。 「同僚を招待すると14日延長」「紹介で1ヶ月無料」等の Viral Loop が無い
- **推奨**: §6.9 に「紹介プログラム」セクション追加。1紹介 = 1ヶ月無料、3紹介で永久15%OFF
- **期待効果**: CAC -20%、Viral K-factor 0.15→0.4

## 良い点 (3つ)

1. **状態機械を IA / 色 / アイコン / 文言で統一する設計思想** (§1.1, §3.3, §21.1)
   - これは Sales Tech SaaS としては Outreach / Salesloft 級の完成度。一般的な日本語SaaSはここを軽視するため、UX一貫性が無い。状態遷移を IA に揃えたのは設計者の力量を感じる。
2. **HITL × 安全モード × ウォームアップ の「三重防御」UX** (§9, §10, §5.6 F-06)
   - LinkedIn BAN リスクの吸収を「機能の見せ方」で実現しており、規制業界 SaaS としての信頼を醸成できる。 Defensible な PoD のひとつになりうる。
3. **AI出力に必ず "編集 / 採用しない / 根拠を見る" の3操作** (§9.2)
   - 「AI Trust UX」のベストプラクティス。多くの SaaS が「採用」しか提供しないが、 "採用しない" を可視化することでユーザー学習データが取れ、長期的なAI品質向上ループが回る。Outreach の Smart Email が後追いした設計と同等。

## ビジネスインパクト観点 MUST FIX 3件 (Phase 1 着手前に必須)

### MUST FIX 1: プライシング体系の3階層化 + 無料トライアル + 年契約 + 休止プラン (= H3)
- **なぜ最重要**: 現行 1プラン¥48,000のみでは **Self-Serve流入が成立せず CAC が膨らむ**。 ARR / NRR / Churn の3指標すべてが事業KPIとして測定不能になる。
- **やること**: Solo / Team / Scale の3階層、14日無料トライアル、年契約-17%、休止プラン (¥0、データ保持90日) の4点セットを §6.8 に明記。
- **期待効果**: ARR +25%、トライアル流入 +3倍、Net Churn -8pt

### MUST FIX 2: HubSpot 連携を Phase 1 に昇格 (= H4)
- **なぜ最重要**: 主要ペルソナP1 (VPoS) の購買決定基準を満たさないMVP は **Phase 1リリース時点で受注機会の60%を捨てる** ことになる。
- **やること**: §19 ロードマップを更新し、 S12 のうち「HubSpot 一方向 (Lead → Contact 作成)」のみを Phase 1 に含める。Salesforce / Pipedrive / 双方向は Phase 2 維持。
- **期待効果**: P1ペルソナの早期受注、ACV +¥120,000/年、商談獲得率 +30%

### MUST FIX 3: North Star Metric / Activation Metric の定義とオンボ復帰導線 (= H1 + H5)
- **なぜ最重要**: アクティベーションが定義されない SaaS は **データドリブンに改善できず、 Product-Led Growth が回らない**。 D7/D30 リテンションは事業の生命線で、 北極星指標が無いとロードマップの優先順位が永続的に正しくならない。
- **やること**:
  - North Star: `週次 Reply received per active LinkedIn account`
  - Activation: `Sign-up → 7日以内 初REPLIED発生`
  - S03 にオンボチェックリスト (Day 0–14)、 Day 1/3/7 の Lifecycle メール、 LinkedIn接続失敗時の専用リカバリ画面
- **期待効果**: D7リテンション +15〜25pt、オンボ完了率 +15pt、トライアル→有料転換 +10pt
