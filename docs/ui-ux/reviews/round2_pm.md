## NEW HIGH に至らない範囲の MEDIUM 指摘 (Round2)

### NM1. 休止プラン の月額50%設計の根拠不明 (Round1からの後退)

- Round1指摘: ¥0 / 月額 / データ保持90日 / 受信のみ
- v1.1: ¥0 → 「月額50%」+ Solo は休止対象外 への変更
- **問題**: 「忙しい月だけ休止したい」ユーザーは月額50%でも継続コスト負担を重く感じる。 Round1 の「churn防止」効果が想定の 60-70% 程度に減衰する見込み。 Solo (=価格感度高い P3 起業家) が休止対象外なのも矛盾。
- **推奨**: §22.1 に注記列を追加し「休止プラン価格の根拠」を明記、もしくは「¥0 で 90 日 / その後 50% で 180 日 / 解約」の二段階設計に。

### NM2. §21.4 競合差別化表に価格列・サポート列がない

- Round1指摘: 「価格 と 日本語サポート品質 の2列を追加」
- v1.1: §21.4 は 5観点 (言語/文化/知識基盤/安全/透明性) のみ、価格列なし
- **問題**: 顧客が乗り換えを検討する瞬間 (購買意思決定) に最も重要なのは「価格と提供価値の対比」。 ¥9,800 (Solo) は HeyReach $79 と概ね同等、 Team ¥48,000 は HeyReach $159 (Agency) と同等水準で「日本語と安全モード」を理由に選ぶ価値がある —— **これを§21.4で言語化しないと営業現場が苦労する**
- **推奨**: §21.4 に「月額」「日本語 UI / 文書」「日本語 LinkedIn メッセージ品質」「サポート言語」「データ保管リージョン (JP)」の5列を追加。

### NM3. ペルソナ別 LTV/CAC/ARPA の数値仮説 未着手

- Round1 M1 指摘: 「P1〜P4 各ペルソナに `想定ARPA / 想定LTV / 想定CAC / 想定LTV:CAC比` を追記」
- v1.1: §22.1 で ペルソナと価格は紐付くが、 LTV や CAC は無し
- **影響**: GTM (Go-to-Market) で「どのペルソナを最初に獲得するか」が定量判断できない。 P3 (起業家、Solo ¥9,800) と P1 (VPoS、Team ¥48,000) のどちらを優先すべきかの根拠が不在。
- **推奨**:
  ```
  P1 田中 (VPoS):    ARPA ¥48,000-148,000 / LTV ¥1.4M / CAC ¥350K / LTV:CAC 4.0x / Sales-led
  P2 佐藤 (SDR):     チームシート単位、決定者ではない
  P3 山本 (1人代理店): ARPA ¥9,800 / LTV ¥176K / CAC ¥30K / LTV:CAC 5.9x / Self-serve
  P4 鈴木 (情シス):   購買決定者、Scale ¥148,000 への昇格を導く / LTV:CAC 3.5x
  ```
  を §2.1 各ペルソナ末尾に追記。

### NM4. 業界ベンチマーク表示の未着手 (Round1 M4)

- Round1 推奨: KPI カードに「業界中央値 / Top 10%」を併記
- v1.1: §16.2 で計算式は明記されたが、ベンチマーク列なし
- **影響**: Sales Tech SaaS (Outreach, Salesloft) の中核価値である「自分の数値が業界平均と比べて高いか低いか」が表示されない。
- **推奨**: §16.2 KPI数式の隣に「業界ベンチマーク Phase2: 匿名統計から算出」セクションを追加。 Phase1 では「先行データ取得期間」とし、Phase2 で投入計画を明記。

### NM5. ナレッジ (§13) の効果検証計画 未着手 (Round1 M5)

- Round1 推奨: 「Phase 1で5社クローズドベータ → ナレッジあり/なし で返信率 +X% を検証」
- v1.1: §13 改訂なし (品質バッジ 🟢🟡🔴 のまま)
- **影響**: PoD (Point of Difference) の核心である「会社特化 RAG」の **数値根拠がない** ため、 §22 価格 (Team ¥48,000) のValueの 30〜50% が「期待値で売る」状態。
- **推奨**: §13 末尾に「期待効果」セクション追加 ——「クローズドベータ 5社で返信率 +5〜8pt を仮説検証 / 結果は §16 KPI ダッシュに反映」
