## v1.2 で生じた新規問題

### NEW MEDIUM 1 (M-r3-1): 休止プラン の定義が §1.1.2 と §22.1 で食い違う

- §1.1.2 (line 89): 「休止プラン: **月額 ¥0** / 50% コスト保留 / データ 90 日保持 / 受信のみ継続」
- §22.1 (line 1608): 「休止プラン: Team/Scale で **月額 50% 保留** / データ 90 日保持 / 受信のみ」

**問題**: 「月額¥0」と「月額50%」は **正反対の意味**。§1.1.2 の「¥0 / 50% コスト保留」も日本語として曖昧 (¥0なのか50%なのか)。Round2 NM1 の指摘 (休止プラン価格の根拠) を一部修正しようとした際の **校正漏れ** と推定。

**影響**: 営業資料・LP で価格を引用する際にどちらを採用するか決められない。顧客問い合わせの最初の質問になる可能性。

**推奨修正 (10分)**:
- §1.1.2 line 89 を §22.1 と一致させる: 「休止プラン: 月額50% / データ 90 日保持 / 受信のみ継続。Team/Scale のみ」
- もしくは「Day 30 で休止プラン提案」のステップ自体を §22.1 に統合し §1.1.2 では「§22.1 参照」とのみ書く

### NEW MEDIUM 2 (M-r3-2): A/B テスト UI の Phase 帰属が §6.2 と §19 で食い違う (Round2 L6-r2 未解消)

- §6.2 Step4 (line 548): 「A/B テスト [✓ 有効化] 配分 50/50」 — Phase 1 のキャンペーン作成ウィザード仕様として記述
- §19 Phase 2 (line 1509): 「ダーク、英語版、A/B テスト UI」と明記

**問題**: BLOCKER 1 (CRM) と同種の Phase 二重定義。§6.2 がキャンペーン作成ウィザード Step4 の必須UI として記述されているため、Phase 1 のスコープに含まれると誤読される。CRM は綺麗に整合させたのに **A/B が見落とされた**。

**影響**: 軽度。CRMほど営業上の致命傷にはならないが、Phase 1 のスプリント計画で A/B 機能を「やる/やらない」で議論が再燃するリスク。

**推奨修正 (5分)**: §6.2 Step4 末尾に「(A/B テスト UI は Phase 2 / Phase 1 は単一案のみ)」と注記、または §19 Phase 1 リストに A/B テストを明示昇格。
